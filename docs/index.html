How it works

Three steps from ambiguity to action.

01

Input anything

Drop in an idea, a task description, a bug report, or a product concept. ActionForge auto-detects the type — no categorization required.

/actionforge I want to build a SaaS for QR code analytics
02

Select a mode

Optionally prefix your input with a mode to shape vocabulary and structure for your audience — developer, PM, founder, or Jira team.

/actionforge /mode startup build a SaaS for invoicing
03

Act on the output

Receive a structured plan with tasks, acceptance criteria, hypotheses, or Jira issues — ready to copy, assign, and execute immediately.

Everything you need to go from fuzzy to focused

Six output types, each tuned for a specific audience and context.

Task Breakdowns

Dependency-ordered task lists with rationale for every step. Each task is executable, not aspirational.

MVP Plans

Startup-mode plans that lead with the riskiest assumption and a validation method — before you write a line of code.

Jira-Ready Tasks

Full issue blocks with type, epic, priority, story points, Given/When/Then acceptance criteria, and sub-tasks.

Debugging Flows

Confidence-ranked hypotheses, reproduction steps, diagnostic checklists, and per-hypothesis fix proposals.

AI Architecture Plans

Model role definitions, prompt architecture, evaluation rubrics, failure modes, and cost/latency estimates.

Product Roadmaps

User stories, MoSCoW prioritization, acceptance criteria, KPIs, and phase-by-phase roadmaps for PMs.

Six modes. One skill.

Each mode adapts vocabulary, structure, and depth for its target audience.

Mode Audience Focus
dev Developers Architecture, files, implementation steps, edge cases, tests
product Product managers User stories, acceptance criteria, MoSCoW priority, KPIs
startup Founders, indie hackers Validation first, MVP scope, monetization, first customers
debug Developers Ranked hypotheses, reproduction, diagnostic checks, fixes
ai ML engineers Model design, prompt architecture, evaluation, failure modes
jira Scrum teams Full Jira issue blocks with all fields and sub-tasks

Get started in 30 seconds

Three installation paths depending on your setup.

Recommended

npm CLI

Install globally and run the automated installer.

npm install -g @actionforge/claude-skill

The installer copies the skill to ~/.claude/skills/actionforge/ automatically.

Claude Code Marketplace

If available in the plugin marketplace:

/plugin marketplace add zenoproduction/actionforge

Browse and install from within Claude Code's plugin interface.

Manual

Clone and copy the skill directory.

git clone https://github.com/zenoproduction/actionforge.git cp -r actionforge/.claude/skills/actionforge \ ~/.claude/skills/actionforge

Then use it in any Claude Code session:

/actionforge I want to build a tool that helps developers track which parts of their codebase are tested
/actionforge /mode jira the checkout flow breaks on mobile when cart has more than 10 items
/actionforge /mode startup I want to build a SaaS that helps freelancers track invoices
